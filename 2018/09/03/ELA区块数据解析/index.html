ELA区块数据解析

  • 区块是区块链的核心，区块中记录了区块高度、区块时间、挖矿难度、交易等重要信息。
    ELA是elastos系统的主链，采用UTXO模型，其区块由Header及Transaction两部分构成。

Table 1. The struct of the block

FieldDescription
Header区块头，记录区块高度、时间、挖矿难度等
Transaction记录区块内包含的交易信息

Header

  • 区块头中包含了区块的版本、前序区块Hash、交易Merkle Root、表示区块时间的时间戳、挖矿难度要求、区块高度，以及联合挖矿等信息。

Table 2.The struct of the block header

SizeFieldDescription
4 bytesVersion区块版本
32 bytesPrevious前序区块Hash
32 bytesMerkle Root区块中所有交易的MerkleRoot的Hash值
4 bytesTimestamp区块时间
4 bytesBits区块难度要求
4 bytesNonce区块工作量证明
4 bytesHeight区块高度
VariableAuxPOW联合挖矿信息

由于ELA采用与BTC联合挖矿的方式产生区块，所以联合挖矿的相关信息主要在AuxPOW字段中。

AuxPOW

AuxPOW中包含BTC的CoinBase交易、BTC区块头Hash、BTC CoinBase所在Merkleroot的分支、ELA区块Hash所在Merkleroot的分支、BTC区块头等信息

Table 3.The struct of the AuxPOW

SizeFieldDescription
VariableparCoinBaseTxBTC CoinBase交易
32 bytesparentHashBTC 区块头Hash
VariableparCoinBaseMerkle_countBTC CoinBase交易的证明路径信息，为包含Merkle Root中的相关交易Hash数量
VariableparCoinBaseMerkleBTC CoinBase交易的证明路径信息，为Merkle Root相关交易Hash
4 bytesparMerkleIndexBTC CoinBase交易的证明路径信息，记录相关交易Hash的位置
VariableauxMerkleBranch_countELA区块Hash的证明路径信息，为ELA区块Merkle Root相关辅链区块Hash的数量
VariableauxMerkleBranchELA区块Hash的证明路径信息，为Merkle Root相关区块Hash
4 bytesauxMerkleIndexELA区块Hash的证明路径信息，记录相关区块Hash的位置
VariableparBlockHeaderBTC区块头

Transaction

  • 交易是ELA最重要的部分，所有ELA的产生、转移及销毁都包含在不同类型的交易之中。交易中包含输入、输出等信息，所有交易被包含在区块中，最后构成完成的账本。
    ELA的区块大小为8M，由于每个交易的大小不同，所以区块中包含交易的最大数量也是变化的。

Table 4.The struct of the block transaction

SizeFieldDescription
4 bytesTransaction Count区块中包含的交易数量
VariableTransaction Data交易信息
  • 交易由交易类型、花费的UTXO、输出地址及金额、解锁脚本、锁定脚本等组成。

Table 5.The struct of the transaction data

SizeFieldDescription
1 byteTransaction Type交易的类型
1 bytePayload VersionPayload版本
VariablePayload根据交易类型的不同，Payload具有不同的数据结构
1-9 bytes(Variable)Attribute Count属性数量
VariableAttribute交易的属性
1-9 bytes(Variable)Input Count输入的数量，即该交易消耗的UTXO数量
VariableInputs交易花费的UTXO
1-9 bytes(Variable)Output Count输出的数量，即该交易的新产生的UTXO数量
VariableOutputs交易的输出
4 bytesLocktime交易锁定时间，以区块高度高度，用于配合Output中的Output Lock验证该UTXO是否可用
1-9 bytes(Variable)Program CountProgram数量，即解锁脚本及锁定脚本的数量
VariableProgram Data解锁脚本及锁定脚本

Transaction Type

ELA的交易类型主要由以下7类，支持包括主链转账、跨链转账的交易。

Table 6. Transaction Type

TypeDescription
0COINBASE，创币交易
1REGISTERASSET，注册资产
2TRANSFERASSET，转账交易
5SIDECHAINPOW，侧链挖矿交易
6RECHARGETOSIDECHAIN，侧链充值交易，发生在侧链
7WITHDRAWFROMSIDECHAIN，侧链提币交易，发生在主链
8TRANSFERCROSSCHAINASSET，跨链交易

Attribute

Table 7. The struct of the attribute in the transaction

SizeFieldDescription
1 byteUsage
1-9 bytes(Variable)Data Length表示后面数据的长度
VariableData

Input

Input由UTXO组成，其中包括Txid、Index、Sequence等信息。

Table 8.The struct of the transaction input

SizeFieldDescription
32 bytesTxid前序交易Hash
2 bytesIndex在前序交易中的Index
4 bytesSequence挖矿交易中为0xffffffff，其他交易为0x00000000

Output

Output由AssetID、转账金额、Output Lock及表示收币地址的ProgramHash组成。

Table 9.The struct of the transaction output

SizeFieldDescription
32 bytesAssetID表示资产的Hash，每种Asset的AssetID相同
8 bytesValue以sela表示的转账金额
4 bytesOutput LockUTXO锁，用于锁定该UTXO，使其在某高度前不能被花费
21 bytesProgramHash可以转换为地址

Program

Program由UTXO对应的解锁脚本及锁定脚本组成。

Table 10.The struct of the transaction program

SizeFieldDescription
1-9 bytes(Variable)Parameter Length解锁脚本长度
VariableParameter解锁脚本
1-9 bytes(Variable)Code Length锁定脚本长度
VariableCode锁定脚本

  1. 根据区块Hash获取区间原始数据，这里需要使用RPC接口的getblockhash方法

开始解析区块数据之前，需要说明2点：
1) 区块每个字段的数据以小端序(little-endian)的方式存储
2) 对长度变化分为很大的字段，通过 1-9 bytes标定字段内容的长度

buf[0]Description
0xfdbuf[1:3]表示字段长度
0xfebuf[1:5]表示字段长度
0xffbuf[1:9]表示字段长度
elsebuf[0]表示字段长度
